# mojo-audio Roadmap

**Last updated:** 2026-03-21
**Context:** Building the DSP + ML inference layer for a local, offline voice conversion
pipeline (Shade) that runs natively on NVIDIA DGX Spark SM_121 ARM64.

---

## North Star

> The entire Shade voice conversion pipeline — stem separation → content extraction →
> pitch estimation → synthesis → output — runs end-to-end on DGX Spark with GPU acceleration,
> zero PyTorch CUDA dependency, zero cloud.

## Sprint 3 — "HiFiGAN Vocoder" ✅ COMPLETE

**Goal:** Implement the NSF-HiFiGAN neural vocoder in MAX Graph.
Takes 192-dim latent features + F0 pitch → audio waveform. Supports 32k/40k/48k.

### What was built
- Full NSF-HiFiGAN generator as MAX Graph
- Weight loader for RVC v2 `.pth` checkpoints (weight-norm reconstruction, BatchNorm baking)
- PyTorch reference implementation for numerical validation
- 16 tests (15 pass, 1 xfail for batch>1)

### MAX Engine bugs discovered and worked around
- `modular/modular#6248` — `ops.conv2d` incorrect for C_in >= 8
- Multi-stride `ops.conv2d` compiler crash (unfiled — needs repro)
- `ops.rebind` overwrites static dimensions (unfiled — needs repro)
- All conv2d usage replaced with im2col + matmul workaround

### Definition of Done — Sprint 3
- [x] HiFiGAN inference in MAX Graph — neural filter matches PyTorch at 0.9998 correlation
- [x] Real RVC checkpoints load and produce valid output (tested: Weeknd 48k, Drake 40k)
- [ ] Runs on Spark GPU (CPU verified, GPU not yet tested)
- [ ] Spark GPU benchmark

### Key files
- `src/models/hifigan.py` — Public NSFHiFiGAN API
- `src/models/_hifigan_graph.py` — MAX graph builder (all im2col, no conv2d)
- `src/models/_hifigan_weight_loader.py` — RVC checkpoint loader

## Sprint 4 — "Full VITS Synthesis" ✅ COMPLETE

**Goal:** Build the remaining RVC v2 inference components (enc_p, flow, speaker embedding)
and wire them with existing AudioEncoder, RMVPE, and HiFiGAN into a single VoiceConverter API.

### What was built

- **TextEncoder (enc_p)** — 6-layer transformer with relative positional attention in MAX Graph
- **Normalizing flow** — 4-stage reverse ResidualCouplingBlock with WaveNet + speaker conditioning
- **Speaker embedding** — baked into HiFiGAN's conv_pre.bias at load time
- **VoiceConverter** — orchestrator class: `VoiceConverter.from_pretrained(ckpt).convert(audio)`
- **PyTorch reference** — enc_p + flow + full VITS inference for numerical validation

### Key decisions
- Relative position attention biases precomputed in numpy (MAX symbolic T limitation)
- Speaker embedding baked as constant (single-speaker models)
- FAISS index retrieval deferred to post-Sprint 4
- Flip channels via permutation matrix matmul (avoids OOM from per-channel slicing)

### Definition of Done — Sprint 4
- [x] enc_p matches PyTorch < 1e-3
- [x] flow matches PyTorch < 1e-3
- [x] Speaker conditioning baked and verified
- [x] VoiceConverter API with convert() method
- [x] 49 tests pass on Spark + all regression tests green
- [ ] Full end-to-end integration test: WAV in → WAV out (needs AudioEncoder + RMVPE loaded simultaneously)
- [ ] Runs end-to-end on DGX Spark with GPU
- [ ] Quality comparison vs Applio output

### Key files
- `src/models/voice_converter.py` — Public VoiceConverter API
- `src/models/_vits_graph.py` — MAX graphs for enc_p + flow

## Sprint 6 — "Multi-Spark" (Post-Shade Launch)

**Goal:** NVLink tensor parallelism across paired Sparks.

### What to implement

- Shard transformer layers across 2 NVLinked Sparks using `max.nn.DistributedTransformer`
- Target: VITS synthesis sharded across 2 GPUs (256GB unified memory pool)
- Enable batch processing of multiple voice conversions simultaneously

### When to do this

After Shade is working on a single Spark. The multi-Spark work is for scale —
handling concurrent requests from multiple clients. For the initial demo, single-Spark
is sufficient.
